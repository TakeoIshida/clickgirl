# ClickGirl — ゲーム仕様書

## 概要

女の子キャラクターをタップして売上を稼ぎ、社員を採用・育成しながら会社を大きくしていくクリッカー系放置ゲーム。

- **プラットフォーム**: iOS
- **技術**: Swift / SpriteKit
- **通貨単位**: 円（¥）

## 画面レイアウト（GameScene）

### 縦方向の領域分割

```
┌─────────────────────┐ ← frame.height
│  HUD（120pt）        │   所持金・収益表示
├─────────────────────┤ ← frame.height - 120
│                     │
│  メインキャラ表示     │   _nobg PNG を使用
│  幅: frame.width×0.85│   HUDと社員パネルの間に縦中央配置
│  高さ: アスペクト比自動│   最大高さ=(エリア高さ×0.95)で自動縮小
│                     │
├─────────────────────┤ ← y = 210
│  オフィスちびキャラ   │   chibiW=36pt（縦長画像対応サイズ）
│  (layerH=120pt)     │
├─────────────────────┤ ← y = 210
│  社員パネル（210pt） │   5キャラのカード表示
└─────────────────────┘ ← y = 0
```

### 社員カード画像表示

- 画像: 背景ありJPG（`employee.imageName`）を使用
- 幅: 最大138pt
- 高さ: アスペクト比維持、**最大110pt**（超過時は幅を自動縮小）
- 縦長画像（りお 784×1176）は幅が自動的に縮小される

## 収益システム

### タップ収益

```
タップ値 = (1.0 + 採用済み社員の収益/秒の合計 × 0.1) × tapMultiplier
```

### 自動収益（毎秒）

```
totalIncomePerSec = (社員収益合計 × incomeMultiplier × officeIncomeMultiplier × floorIncomeMultiplier) + cityIncomePerSec
```

### コンボシステム

- 0.85秒以内に連続タップするとコンボカウント増加
- コンボボーナス: `+10% × (コンボ数 − 1)` （上限 +100%）
- 0.85秒タップなしでリセット

## ガチャ

### コスト

| 種別 | コスト |
|------|--------|
| シングル | ¥50,000 |
| 10連 | ¥450,000（1回分お得） |

### レアリティ・排出率

| レアリティ | 排出率 | 収益ボーナス（カード1枚あたり） |
|-----------|--------|-------------------------------|
| N | 60% | +2% |
| R | 30% | +5% |
| SR | 8% | +15% |
| SSR | 2% | +40% |

### 天井

- 10回ごとに **R以上** が確定

### カード演出（めくり）

ガチャ結果はカードの裏面が表示された状態で始まり、プレイヤーが能動的にめくって公開する。

#### 1枚引き
1. カード裏面がポップインで登場
2. ヒント「タップしてめくろう！」が表示される
3. カードをタップ → フリップアニメーションで表面を公開
4. SR/SSR の場合はフラッシュエフェクトが発動
5. ヒントが「タップで閉じる」に切り替わる

#### 10連引き
1. 10枚の裏面が左上から順に波状にポップイン（0.08秒刻み）
2. 「✨ 全てめくる」ボタンと「カードをタップしてめくろう」ヒントが表示される
3. **個別めくり**: 各カードをタップすると1枚ずつフリップ
4. **まとめてめくる**: 「全てめくる」ボタンをタップすると残りを左から順に連続フリップ（0.12秒刻み）
5. 全カードめくり完了後、「全てめくる」ボタンが消えてヒントが「タップで閉じる」に切り替わる

#### カード裏面デザイン
- 深紫背景 + 紫ボーダー、回転する ✨ アイコン、「CLICK GIRL」テキスト（小カードは「CG」）

#### フリップアニメーション仕様
- 前半: 裏面の xScale が 1 → 0（easeIn、0.13秒）
- めくり瞬間: 白いシマー（光）が瞬く
- 後半: 表面の xScale が 0 → 1.08 → 1.0（easeOut + バウンス、0.18秒）

### 10連グリッドレイアウト

**配置: 3+3+3+1**（3列×3行 + 最後の1枚を画面中央）

カードサイズを画面幅・高さから動的に計算し、4行が利用可能領域に収まるよう上限クランプを適用する。

```
cardW = floor((画面幅 - 24 - 10×2) / 3)
cardH = min(floor(cardW × 1.58), floor((利用可能高さ - 8×3) / 4))
```

| 画面幅 | カードサイズ（目安） |
|--------|------------------|
| 390pt（iPhone 14等） | 115 × 160 pt |
| 375pt（iPhone SE等） | 110 × 153 pt |

- 余白: 左右各 12pt、カード間 10pt、行間 8pt
- 縦位置: 「全てめくる」ボタン領域（下115pt）と上部マージン（65pt）の間でグリッド全体を縦中央寄せ
- 最終10枚目は単独で画面横中央に配置
- フォント・バッジ・NEWラベルはカードサイズに比例してスケール

### カード収益倍率

同じキャラのカードを複数枚所持するほど、そのキャラの収益に倍率が加算される。

```
cardIncomeMultiplier = 1.0 + Σ(rarity.incomeBonus × 所持枚数)
```

## オフィス

### フロア拡張

- フロアを増やすごとに自動収益 **+30%**
- フロア拡張コスト: `80,000 × 5^(現フロア数 − 1)`

### オフィス設備

各種設備をアップグレードすることで自動収益にボーナスが加算される。

```
officeIncomeMultiplier = 1.0 + Σ(level × incomeBoostPerLevel)
```

---

## 都市（シティ）

- 空きプロットに建物を建設し、毎秒の収益を追加できる
- 建物はレベルアップが可能（最大レベルあり）
- 都市収益は `totalIncomePerSec` に直接加算される

---

## 放置システム

- アプリを閉じている間も社員が自動収益を稼ぎ続ける
- 放置上限: **8時間**（「夜勤延長」購入で **16時間**）
- 再起動時にオフライン収益をポップアップで通知

## 収益倍率の適用順序

```
totalIncomePerSec
  = (Σ 社員収益/秒 × cardIncomeMultiplier)
      × incomeMultiplier      // ショップ永続 + 時限ブースト
      × officeIncomeMultiplier // オフィス設備
      × floorIncomeMultiplier  // フロア数
  + cityIncomePerSec           // 都市収益（倍率対象外）
```
